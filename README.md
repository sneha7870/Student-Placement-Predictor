# Student Placement Predictor Using Flask & Docker

A web application that takes a student's academic and extracurricular record
as numeric input (CGPA, attendance, backlogs, internships, etc.) and uses a
trained Machine Learning model (Random Forest Classifier) to predict whether
the student is likely to be **Placed** or **Not Placed**, along with a
confidence score.

## Features

- Clean web form that only accepts numeric input, with min/max validation
  for every field (CGPA, 10th %, 12th %, attendance, backlogs, internships,
  projects, communication skill, coding/aptitude score)
- Server-side validation with clear error messages for invalid input
- ML model (scikit-learn `RandomForestClassifier`) trained on a generated
  dataset that models realistic relationships between academics/experience
  and placement outcome
- Prediction result shown with a **Placed / Not Placed** label and a visual
  confidence gauge
- Fully containerized with **Docker** for consistent deployment
- Deployable on **Render** (or any platform supporting Docker/Gunicorn)

## Run Locally (without Docker)

```bash
# 1. Clone the repo
git clone https://github.com/<sneha7870>/student-placement-predictor.git
cd student-placement-predictor

# 2. Create a virtual environment
python -m venv venv
source venv/bin/activate      # on Windows: venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Train the model (creates model.pkl)
python train_model.py

# 5. Run the app
python app.py
```

Visit **http://localhost:5000** in your browser.

## Run with Docker

```bash
# Build the image (also trains the model during build)
docker build -t placement-predictor .

# Run the container
docker run -p 5000:5000 placement-predictor
```

Visit **http://localhost:5000**.

## How the Prediction Works

1. `train_model.py` generates a dataset where each student's placement
   likelihood is computed as a weighted combination of their CGPA, academic
   percentages, attendance, backlog count, internships, projects,
   communication skill, and coding score — plus random noise, to simulate
   realistic variation.
2. A `RandomForestClassifier` is trained on this data and saved to
   `model.pkl` using `joblib`.
3. `app.py` loads `model.pkl` at startup. When a user submits the form,
   the input is validated, converted into the same feature format the
   model was trained on, and passed to `model.predict()` /
   `model.predict_proba()`.
4. The result — **Placed** or **Not Placed**, plus a confidence percentage —
   is rendered back on the page.

> **Note:** The model in this repository is trained on synthetically
> generated data for demonstration purposes. For production use, replace
> `train_model.py`'s dataset generation with a real historical placement
> dataset from your institution for more accurate predictions.
